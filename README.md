# Job Tracking: REST API & CLI сервис для подбора вакансий

[![Java](https://img.shields.io/badge/Java-17%2B-blue.svg)](https://www.oracle.com/java/)
[![Spring Boot](https://img.shields.io/badge/Spring%20Boot-3.3.3-brightgreen.svg)](https://spring.io/projects/spring-boot)
[![License](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

**Job Tracking** — это универсальный бэкенд-сервис для автоматизации процесса подбора вакансий. Проект прошел путь эволюции от автономного консольного приложения до современного **REST API на базе Spring Boot 3.3.3**. Архитектура сервиса позволяет гибко управлять данными, используя реляционную базу данных и интеллектуальные алгоритмы оценки релевантности.

Приложение поддерживает два режима работы: веб-интерфейс (REST) и встроенный консольный интерфейс (CLI).

---

## 🚀 Режимы работы

### 1. REST API (Основной режим)
Приоритетный режим работы. При запуске активируется сервер Tomcat (порт 8080). Данные хранятся в БД H2 (In-memory).
- **Точка входа:** `ru.vk.education.job.JobApplication`
- **Команда для запуска:** `./gradlew bootRun`
- **H2 Console:** `http://localhost:8080/h2-console` (JDBC URL: `jdbc:h2:mem:jobdb`)

### 2. Консольный режим (CLI)
Интерактивный режим внутри Spring-контекста. Позволяет управлять системой через терминал сразу после запуска приложения.
- **Особенности:** Каждая введенная команда сохраняется в `history.txt` через `FileService`. При перезапуске система может автоматически считывать историю (зависит от настроек лога).

## 🛠️ Технологический стек

- **Java 17**: Использование Records и современного Stream API.
- **Spring Boot 3.3.3**: Ядро системы, управление зависимостями и бинами.
- **Spring JDBC**: Использование `NamedParameterJdbcTemplate` для безопасной работы с SQL.
- **H2 Database**: Реляционная база данных, работающая в оперативной памяти.
- **Lombok**: Минимизация шаблонного кода (геттеры, сеттеры).
- **Gradle**: Сборка и автоматизация.

## 🔌 REST API Справочник

### Управление пользователями (`/users`)

| Метод  | Эндпоинт            | Описание                      | Тело запроса (JSON)                                                |
|:-------|:--------------------|:------------------------------|:-------------------------------------------------------------------|
| `POST` | `/users`            | Регистрация нового кандидата  | `{"name": "david", "skills": ["java", "spring"], "experience": 5}` |
| `GET`  | `/users`            | Получить список всех профилей | -                                                                  |
| `GET`  | `/users/{username}` | Поиск пользователя по имени   | -                                                                  |

### Управление вакансиями (`/jobs`)

| Метод  | Эндпоинт | Описание                     | Тело запроса (JSON)                                                                |
|:-------|:---------|:-----------------------------|:-----------------------------------------------------------------------------------|
| `POST` | `/jobs`  | Добавление новой вакансии    | `{"title": "Senior Java Dev", "company": "VK", "tags": ["java"], "experience": 3}` |
| `GET`  | `/jobs`  | Список всех открытых позиций | -                                                                                  |

## 💻 CLI Справочник команд

| Команда     | Описание              | Пример                                                |
|:------------|:----------------------|:------------------------------------------------------|
| `user`      | Создать пользователя  | `user david --skills=java,spring --exp=5`             |
| `job`       | Создать вакансию      | `job Java-Developer --company=VK --tags=java --exp=3` |
| `user-list` | Показать всех юзеров  | `user-list`                                           |
| `job-list`  | Показать все вакансии | `job-list`                                            |
| `suggest`   | Запустить подбор      | `suggest david`                                       |
| `stat`      | Вывод статистики      | `stat --top-skills 5` или `stat --exp 3`              |
| `history`   | Просмотр лога команд  | `history`                                             |
| `exit`      | Завершить работу      | `exit`                                                |

## 🏗️ Структура проекта

- **`controller`**: Эндпоинты для API (Users, Jobs, Stats, Suggest).
- **`service`**: Бизнес-логика подбора (`JobService`, `JobMatch`).
- **`repository`**: Слой доступа к БД через JDBC (`UserRepository`, `JobRepository`).
- **`domain`**: Сущности данных (`User`, `Job`).
- **`util`**: Вспомогательные компоненты (`FileService` для CLI, `JobMatchTask` для фоновых задач).
