# Job Tracking: Консольный сервис для подбора вакансий

Это пет-проект, представляющий собой консольное Java-приложение для симуляции бэкенда сервиса по подбору вакансий. Приложение позволяет создавать профили пользователей и вакансии, а также подбирать наиболее подходящие вакансии для пользователей на основе их навыков и опыта.

## Ключевые возможности

- **Управление пользователями:** Создание профилей пользователей с указанием их навыков (skills) и опыта (experience).
- **Управление вакансиями:** Добавление вакансий с названием, компанией, требуемыми навыками (tags) и опытом.
- **Подбор вакансий (`suggest`):** Ручной запуск подбора двух наиболее релевант-ных вакансий для конкретного пользователя.
- **Фоновый подбор:** Автоматический периодический запуск задачи (`JobMatchTask`) для поиска совпадений между пользователями и вакансиями.
- **Статистика:** Получение статистики по вакансиям и навыкам (например, самые востребованные навыки).
- **Сохранение состояния:** Вся история команд сохраняется в файл, что позволяет восстанавливать состояние приложения после перезапуска.

## Как это работает

Приложение построено на нескольких ключевых компонентах:

- **In-memory хранилище:** Все данные (пользователи и вакансии) хранятся в оперативной памяти в экземпляре класса `JobService`. Это обеспечивает бы-стрый доступ, но данные теряются при завершении работы без сохранения.
- **Сохранение истории:** Для персистентности состояния используется простой механизм: каждая введенная команда записывается в файл `history.log`. При старте приложения все команды из этого файла считываются и выполняются заново, восстанавливая данные.
- **Консольный интерфейс:** Взаимодействие с пользователем происходит через стандартный ввод/вывод в консоли. Приложение считывает команды, парсит их и выполняет соответствующую логику.
- **Фоновая задача:** `ScheduledExecutorService` используется для запуска фоновой задачи `JobMatchTask` раз в минуту. Эта задача выполняет логику подбора вакансий для всех пользователей.

## Как запустить

1.  Склонируйте репозиторий на свой компьютер.
2.  Откройте проект в вашей любимой IDE (например, IntelliJ IDEA).
3.  Убедитесь, что у вас установлен JDK версии 8 или выше.
4.  Найдите и запустите метод `main` в классе `ru.vk.education.job.Main`.
5.  После запуска в консоли можно вводить команды.

## Справочник команд

#### Создание пользователя (`user`)
Добавляет нового пользователя в систему.

**Формат:**
```
user <username> --skills=<skill1,skill2,...> --exp=<experience>
```
**Пример:**
```
user ivan_ivanov --skills=java,spring,postgres --exp=5
```

#### Список пользователей (`user-list`)
Выводит список всех зарегистрированных пользователей.

**Формат:**
```
user-list
```

#### Создание вакансии (`job`)
Добавляет новую вакансию.

**Формат:**
```
job <title> --company=<company_name> --tags=<tag1,tag2,...> --exp=<required_experience>
```
**Пример:**
```
job Java-Developer --company=VK --tags=java,spring,docker --exp=3
```

#### Список вакансий (`job-list`)
Выводит список всех добавленных вакансий.

**Формат:**
```
job-list
```

#### Предложить вакансии (`suggest`)
Рекомендует топ-2 вакансии для указанного пользователя.

**Формат:**
```
suggest <username>
```
**Пример:**
```
suggest ivan_ivanov
```

#### Статистика (`stat`)
Показывает различную статистику.

**Форматы:**
- `stat --exp <years>`: Показать вакансии, требующие `>= years` опыта.
- `stat --match <count>`: Показать пользователей, которым подошло `>= count` вакансий.
- `stat --top-skills <count>`: Показать `count` самых востребованных навыков.

**Примеры:**
```
stat --exp 3
stat --top-skills 5
```

#### История команд (`history`)
Показывает все ранее введенные команды.

**Формат:**
```
history
```

#### Выход (`exit`)
Завершает работу приложения.

**Формат:**
```
exit
```

## Структура проекта

- `Main.java`: Главный класс, точка входа. Отвечает за парсинг команд и запуск основных процессов.
- `domain/`: Пакет с моделями данных (`User`, `Job`).
- `service/`: Пакет с бизнес-логикой.
  - `JobService.java`: Основной сервис для управления пользователями и вакансиями.
  - `JobMatch.java`: Класс, представляющий "совпадение" вакансии с пользователем, включая оценку релевантности.
- `util/`: Пакет со вспомогательными утилитами.
  - `FileService.java`: Утилита для чтения и записи в файл истории.
  - `JobMatchTask.java`: Фоновая задача для автоматического подбора вакансий.

## Возможные улучшения

- **База данных:** Заменить in-memory хранилище и файловую историю на полноценную базу данных (например, PostgreSQL или H2).
- **REST API:** Обернуть бизнес-логику в RESTful-сервис с помощью Spring Boot.
- **Веб-интерфейс:** Создать простой UI для взаимодействия с сервисом.
- **Усложнение алгоритма:** Улучшить алгоритм подбора, добавив больше параметров (например, вес навыков, география).
- **Тесты:** Написать юнит-тесты для сервисного слоя и логики подбора.
