# Job Tracking: REST API & CLI сервис для подбора вакансий

[![Java](https://img.shields.io/badge/Java-17%2B-blue.svg)](https://www.oracle.com/java/)
[![Spring Boot](https://img.shields.io/badge/Spring%20Boot-3.x-brightgreen.svg)](https://spring.io/projects/spring-boot)
[![License](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

**Job Tracking** — это универсальный бэкенд-сервис для автоматизации процесса подбора вакансий. Проект прошел путь эволюции от автономного консольного приложения до современного **REST API на базе Spring Boot**. Архитектура сервиса позволяет гибко управлять данными о кандидатах и открытых позициях, используя интеллектуальные алгоритмы оценки релевантности.

Приложение поддерживает два полнофункциональных режима работы: современный веб-интерфейс для интеграции с фронтенд-системами и классический консольный интерфейс (CLI) для локального управления и тестирования.

---

## 🚀 Режимы работы

### 1. REST API (Основной режим)
Это приоритетный режим функционирования приложения как микросервиса. При запуске активируется встроенный сервер Tomcat, предоставляющий программный интерфейс для внешних HTTP-запросов. Данные хранятся в потокобезопасном `in-memory` репозитории.
- **Точка входа:** `ru.vk.education.job.JobApplication`
- **Команда для запуска:** `./gradlew bootRun`
- **Базовый URL:** `http://localhost:8080`

### 2. Консольный режим (CLI / Legacy)
Интерактивный режим, сохраняющий оригинальный опыт работы через терминал. Поддерживает парсинг командных аргументов и обеспечивает персистентность через механизм ведения логов.
- **Точка входа:** `ru.vk.education.job.Main`
- **Особенности:** Каждая команда записывается в файл `history.log`. При перезапуске CLI-режима сервис автоматически считывает историю и восстанавливает состояние всех объектов в памяти.

## 🛠️ Технологический стек

- **Java 17+**: Использование современных рекордов и расширенных возможностей коллекций.
- **Spring Boot 3.x**: Использование Spring Web для маршрутизации и Spring DI для управления компонентами.
- **Java Concurrency**: Асинхронный подбор вакансий реализован через `ScheduledExecutorService` (в консоли) и фоновые задачи `JobMatchTask`.
- **In-memory Storage**: Собственные реализации репозиториев на базе потокобезопасных структур данных.
- **Gradle**: Автоматизация сборки и управления зависимостями.

## 🔌 REST API Справочник

### Управление пользователями (`/users`)

| Метод  | Эндпоинт            | Описание                      | Тело запроса (JSON)                                                    |
|:-------|:--------------------|:------------------------------|:-----------------------------------------------------------------------|
| `POST` | `/users`            | Регистрация нового кандидата  | `{"username": "david", "skills": ["java", "spring"], "experience": 5}` |
| `GET`  | `/users`            | Получить список всех профилей | -                                                                      |
| `GET`  | `/users/{username}` | Поиск пользователя по имени   | -                                                                      |

### Управление вакансиями (`/jobs`)

| Метод  | Эндпоинт | Описание                     | Тело запроса (JSON)                                                                |
|:-------|:---------|:-----------------------------|:-----------------------------------------------------------------------------------|
| `POST` | `/jobs`  | Добавление новой вакансии    | `{"title": "Senior Java Dev", "company": "VK", "tags": ["java"], "experience": 3}` |
| `GET`  | `/jobs`  | Список всех открытых позиций | -                                                                                  |

### Рекомендации и Статистика (`/suggest`, `/stat`)

| Метод | Эндпоинт                         | Описание                                                      |
|:------|:---------------------------------|:--------------------------------------------------------------|
| `GET` | `/suggest/{username}`            | Интеллектуальный подбор топ-2 вакансий для пользователя       |
| `GET` | `/stat/top-skills/{n}`           | Список `n` самых востребованных навыков среди работодателей   |
| `GET` | `/stat/jobs-by-experience/{exp}` | Вакансии с требованием опыта от `exp` лет и выше              |
| `GET` | `/stat/users-by-matches/{count}` | Список кандидатов, имеющих `>= count` совпадений с вакансиями |

## 💻 CLI Справочник команд

| Команда     | Описание              | Пример                                                |
|:------------|:----------------------|:------------------------------------------------------|
| `user`      | Создать пользователя  | `user david --skills=java,spring --exp=5`             |
| `job`       | Создать вакансию      | `job Java-Developer --company=VK --tags=java --exp=3` |
| `user-list` | Показать всех юзеров  | `user-list`                                           |
| `job-list`  | Показать все вакансии | `job-list`                                            |
| `suggest`   | Запустить подбор      | `suggest david`                                       |
| `stat`      | Вывод статистики      | `stat --top-skills 5` или `stat --exp 3`              |
| `history`   | Просмотр лога команд  | `history`                                             |
| `exit`      | Выход из приложения   | `exit`                                                |

## 🏗️ Структура проекта

- **`ru.vk.education.job.controller`**: Обработка HTTP-трафика, десериализация JSON и управление ответами API.
- **`ru.vk.education.job.service`**: Ядро системы. Содержит `JobService` (бизнес-логика) и `JobMatch` (алгоритм ранжирования релевантности).
- **`ru.vk.education.job.domain`**: Доменные сущности `User` и `Job`, описывающие структуру данных.
- **`ru.vk.education.job.repository`**: Слой абстракции над данными в оперативной памяти.
- **`ru.vk.education.job.util`**:
    - `JobMatchTask`: Многопоточный планировщик для фонового поиска совпадений.
    - `FileService`: Обеспечение сохранности истории команд для CLI.
